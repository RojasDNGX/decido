# 🧪 Decido — User Testing Guide

This document defines how to run qualitative user tests for Decido.

---

## 🎯 Objective

Validate:

* clarity of value
* usability without explanation
* trust in the recommendation
* points of friction

---

## 👥 User Profiles

Test with 10–20 users across:

* High workload professionals
* Casual users
* Low-tech users

---

## 🧭 Test Flow

1. User lands on homepage
2. User clicks main CTA
3. User interacts with the product
4. User receives a recommendation

---

## 🗣️ Introduction (Neutral)

"Obrigado por participar. Estamos testando uma nova ferramenta.
Não existem respostas certas ou erradas — estamos testando o sistema, não você.
Use como achar mais natural e, se possível, pense em voz alta."

---

## 🎬 Scenario

"Imagine que você começou seu dia e tem estas tarefas:

* responder e-mails acumulados
* comprar um presente
* terminar um relatório importante para amanhã
* marcar uma consulta médica"

---

## 🧩 Task

"Use a ferramenta para decidir o que fazer primeiro."

---

## 👀 Observation Checklist

* Qual foi a primeira ação?
* Houve hesitação?
* Entendeu onde digitar?
* Entendeu a recomendação?
* Clicou em “Por que?”?
* Reação ao resultado?

---

## 💬 Post-Test Questions

* O que achou da recomendação?
* Ficou confuso em algum momento?
* Confiaria nisso no seu dia a dia?
* O que mudaria?

---

## ✅ Success Criteria

* Usuário completa o fluxo sem ajuda
* Entende o que fazer
* Demonstra confiança mínima na decisão
* Padrões de comportamento identificáveis

---

## 🧠 Rule

Do not guide the user during interaction.

Only observe.
